# AWS Authoritative DNS for ts.t11s.net Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Stand up two AWS `t4g.nano` DNS servers (one in `us-west-2`, one in `us-east-2`) that serve as authoritative-feeling resolvers for the `ts.t11s.net` zone over Tailscale, with records managed in a Route 53 private hosted zone, instances self-bootstrapping from FlakeHub on first boot, and zero operator SSH ever required.

**Architecture:**

- Tailnet clients use Tailscale split-DNS to send `ts.t11s.net` queries to the two boxes' Tailscale 100.x IPs.
- Each box runs CoreDNS bound to `tailscale0`, forwarding queries to the AWS VPC resolver (`169.254.169.253` / `fd00:ec2::253`); the VPC resolver answers from a Route 53 private hosted zone associated with both VPCs.
- Boxes self-bootstrap via user-data on first boot: `determinate-nixd login aws` + `fh apply nixos burk3/all-the-nix/*#nixosConfigurations.<host>`. A 30-minute timer keeps them converged on the latest published release.
- Tailscale auth keys are generated by the `tailscale_tailnet_key` Terraform resource and piped to AWS SSM Parameter Store; instances fetch them via the IAM role assigned at launch.
- Releases publish via GitHub Actions: free x86 runner dispatches `aarch64-linux` builds to nixbuild.net, outputs auto-push to FlakeHub Cache, `flakehub-push` tags the release.

**Tech Stack:** NixOS (aarch64-linux) on Determinate AMI · snowfall-lib · CoreDNS · Tailscale (provider for auth keys) · AWS (Route 53 + EC2 + VPC + IAM + SSM) · Terraform (local state) · GitHub Actions · nixbuild.net · FlakeHub Cache · Determinate Nix.

**Manual prerequisites:** The "Do before execution" steps in [`docs/aws-dns-manual-steps.md`](../../aws-dns-manual-steps.md) must be complete before subagent execution begins (AWS creds, `NIXBUILD_TOKEN` in GitHub secrets, FlakeHub OIDC trust, Tailscale ACL with `tag:dns-node`, Tailscale OAuth client credentials in hand). The "Do after instances are up" step (Tailscale split-DNS config) gets done out-of-band after Phase B applies; Task C3 verifies it landed correctly.

### Task A4: Add `tailscale-bootstrap` service to the module

This service fetches the auth key from SSM on first boot and runs `tailscale up`. Idempotent: skips if tailscale is already authenticated.

**Files:**
- Modify: `modules/nixos/dns-server/default.nix`

- [ ] **Step 1: Append the systemd service inside the `config = lib.mkIf cfg.enable { … };` block**

```nix
    systemd.services.tailscale-bootstrap = {
      description = "Join tailnet using auth key from SSM Parameter Store";
      wantedBy = [ "multi-user.target" ];
      after = [ "tailscaled.service" "network-online.target" ];
      wants = [ "tailscaled.service" "network-online.target" ];
      serviceConfig = {
        Type = "oneshot";
        RemainAfterExit = true;
      };
      path = [
        pkgs.awscli2
        pkgs.tailscale
      ];
      script = ''
        set -euo pipefail
        if tailscale status --json 2>/dev/null | ${pkgs.jq}/bin/jq -e '.BackendState == "Running"' >/dev/null; then
          echo "tailscale already running; nothing to do"
          exit 0
        fi
        AUTH_KEY=$(aws ssm get-parameter \
          --name ${cfg.tailscaleAuthKeyParam} \
          --with-decryption \
          --query 'Parameter.Value' --output text)
        tailscale up \
          --authkey "$AUTH_KEY" \
          --hostname "${config.networking.hostName}" \
          --advertise-tags=tag:dns-node \
          --ssh
      '';
    };
```

# dns-server

Authoritative-feeling DNS for `ts.t11s.net`. CoreDNS binds to `tailscale0` and forwards to the AWS VPC resolver (`169.254.169.253` / `fd00:ec2::253`), which answers from the Route 53 private hosted zone associated with the VPC. Stock CoreDNS only — no plugin builds.

On first boot, `tailscale-bootstrap` fetches a one-time auth key from SSM (`/dns-nodes/tailscale-authkey` by default) using the instance's IAM role and joins the tailnet under `tag:dns-node`.

The `fh-apply` timer pulls the latest `burk3/all-the-nix` release from FlakeHub every 30 minutes and activates it. Tag a release to ship a change; cattle converge within the timer window. No SSH involved.

Pairs with `terraform/` for the AWS side and `tailscale_tailnet_key` for auth-key generation.
